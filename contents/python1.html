コメント

「#」から行末まではコメントとみなされ、Pythonの文とは解釈されずに無視される。
プログラム内に説明や注釈を追加するためのメモのようなもの。 
		
# 素数判定関数
def is_prime(i):
    if i <= 1:                            # 1以下ならば素数でない
        return False
    for j in range(2, int(i**0.5) + 1):   # 順に割って割り切れたら素数でない
        if i % j == 0:
            return False
    return True                           # そうでなければ素数

標準出力と標準入力

最も一般的である、ディスプレイへの出力を標準出力、キーボードからの入力を標準入力という。 Pythonでは、標準出力のための print関数 、標準入力のための input関数 、が用意されている。

print 関数

			
def print(
    *objects: object,
    sep: str = " ",
    end: str = "\n"
) -> None
引数 型 初期値 説明
*objects object 表示したいオブジェクト
sep str " " 区切り文字
end str "\n" 末尾に付ける文字
コンマ区切りで指定された *objects の区切り文字を sep にし、末尾に end を付け加えた文字列を表示する。

※「*」は可変長引数を意味する記号である。任意の数の引数をコンマ区切りで渡すことができる。
※「\n」は改行を意味するエスケープシーケンスである。環境によってはバックスラッシュ「\」が日本円記号「¥」と表示される場合がある。 

			
print("apple")                              # apple
print("app", "le")                          # app le
print("app", "le", sep = "-")               # app-le
print("b", "n", "na", sep = "a")            # banana
print("b", "n", "n", sep = "a", end = "a")  # banana
print("a", "b", "c", sep = "-", end = "!")  # a-b-c!

コレクション

コレクションとは

以下の３つの条件を満たすデータ型をコレクションという。

  1. 複数のオブジェクトを格納できる（in演算子が使える）
  2. 要素を一つずつ取り出せる（for文で繰り返し処理ができる）
  3. 格納してあるオブジェクトの個数を取得できる（len関数を使える）

また、１つ目の条件を満たすデータ型をコンテナ、２つ目の条件を満たすデータ型はイテラブルであるという。

コレクションの分類

コレクション(collection)は主に、シーケンス(sequence)、セット(set)、マッピング(mapping)に分類される。

シーケンス
番号(index)によって要素を取り出せる。
要素に順番があり、重複可能。
セット（集合）
集合のこと。よって要素に順番がなく、重複不可。
マッピング
キー(key)によって値(value)を取り出せる。
要素に順番があり、key は重複不可。


ミュータブル（要素の変更が可能）かどうかによっても分類できる。
ミュータブル：赤色　　　イミュータブル：青色

リスト

リスト(list)は、ミュータブルなシーケンスである。要素をコンマ区切りで表したものを[ ]で囲ったものがリテラルになる。 末尾に余分なコンマが一つ付いていてもよい。
例（リストのリテラル）
  • [2, 3, 5, 7, 11, 13, 17, 19]
  • [-4.5, -0.8, 2.9, 3.8, 4.2,]
  • ['apple', 'banana', 'orange']
  • [2, -3.7, "text", True, "text", 2,]
  • ['a']
  • []

タプル

タプル(tuple)は、イミュータブルなシーケンスである。要素をコンマ区切りで表したものを( )で囲ったものがリテラルになる。 タプルの場合は( )で囲まなくても問題はないが、見やすくなるので通常は囲む。末尾に余分なコンマが一つ付いていてもよい。 単要素のタプルを表す場合には、末尾にコンマが必要。
例（タプルのリテラル）
  • 2, 3, 5, 7, 11, 13, 17, 19
  • (-4.5, -0.8, 2.9, 3.8, 4.2,)
  • 'apple', 'banana', 'orange',
  • (2, -3.7, "text", True, "text", 2)
  • ('a',)
  • ()

文字列

文字列型(str)は、テキストシーケンスとも呼ばれる、イミュータブルなシーケンスである。 例えば、 "apple" は、"a" "p" "p" "l" "e" という５つの要素を格納したものである。 文字列リテラルについては、前に説明したので省く。

range

rangeは、等差型の整数のみを取り扱うイミュータブルなシーケンスである。 リテラルは無いので、後述するrangeコンストラクタを用いることでしか生成できない。 rangeオブジェクトは全ての値を格納しているのではなく、必要に応じてその都度計算して返す。 なので他のコレクションに比べて非常にメモリ消費が少なく、要素の検索が高速である。
例（rangeをリストに変換したもの）
  • [0, 1, 2, 3, 4, 5, 6, 7, 8]
  • [0, 2, 4, 6, 8, 10, 12]
  • [-11, -7, -3, 1, 5, 9, 13]
  • [25, 20, 15, 10, 5, 0, -5]
  • [2]
  • []
※　上に見せているものはあくまでrangeをリストに変換したものなのでリストである。rangeではない。

set

setは、ミュータブルな集合である。要素をコンマ区切りで表したものを{ }で囲ったものがリテラルになる。 末尾に余分なコンマが一つ付いていてもよい。空のsetを表すリテラルはない。後述するsetコンストラクタを用いて、空のsetを生成できる。
例（setのリテラル）
  • {2, 3, 5, 7, 11, 13, 17, 19,}
  • {-4.5, -0.8, 2.9, 3.8, 4.2}
  • {'apple', 'banana', 'orange',}
  • {2, -3.7, "text", True, -8j}
  • {'a'}

frozenset

frozensetは、イミュータブルな集合である。リテラルは無いので、後述するfrozensetコンストラクタを用いることでしか生成できない。

辞書

辞書(dict)は、ミュータブルなマッピングである。「dict」は「dictionary」の略である。「key: value」の組をコンマ区切りで表したものを{ }で囲ったものがリテラルになる。 末尾に余分なコンマが一つ付いていてもよい。
例（辞書のリテラル）
  • {"apple": 180, "peach": 100, "kiwi": 80}
  • {"山田": 90, "小林": 30, "佐藤": 70,}
  • {"India": 14_4170, "China": 14_2520,}
  • {"diabetes": "糖尿病", "pneumonia": "肺炎"}
  • {}

sum関数

def sum(
    iterable: iterable,
    start: number = 0
) -> number
		
			
print(sum([2, 3, 5, 7, 11, 13]))         # 41
print(sum((2, 3, 5, 7, 11, 13)))         # 41
print(sum({2, 3, 5, 7, 11, 13}))         # 41

			print(sum({8:'d9m6', 12:'b3w9', 15:'m2yq'}))  # 35
print(sum({-3: 23, 0: 46, 5: 58, 8: 65}))     # 10

			print(sum([2, 3, 5, 7, 11, 13], start = -5))  # 36

引数に渡したイテラブルオブジェクトについて、次の条件を満たせばTrue、でなければFalseが返される。
関数 条件 空の場合
all 全ての要素がTrue Trueが返される
any いずれかの要素がTrue Falseが返される
※　「要素がTrue」＝「boolコンストラクタを用いて真偽値型に変換したときにTrueになる」
※　「空の場合」＝「引数に渡したイテラブルオブジェクトが空の場合」 
			
print(all([0, 2, 4, 6, 8, 10]))            # False
print(any({0, 2, 4, 6, 8, 10}))            # True
print(all({0, 0.0, 0j, "", None}))         # False
print(any((0, 0.0, 0j, "", None)))         # False
print(all("a b c"))                        # True
print(any("apple"))                        # True
dict の場合は key によって判断される。 
			print(all({5: '', 7: False, 15: 0.0}))     # True
print(any({0: None, 8: "", 12: 0j}))       # True
引数のイテラブルオブジェクトが空の場合はこのようになる。 
			print(all([]))                             # True
print(any({}))                             # False

引数に渡したイテラブルオブジェクトを並び変えた、新たなリストを返す。

    iterable: iterable,
    reverse: bool = False

メモリアドレスは変わらずに、後から値を変更できるものをミュータブル、そうでないものをイミュータブルという。 その違いが明確な言語として、Python や Rust が挙げられる。 変数の値を書きかえる際に、以下のような処理の違いがある。

ミュータブル
メモリアドレスは変わらずに値だけが変更される。
イミュータブル
新たなメモリアドレスに値が用意される。

値渡しと参照渡し

変数から変数への値の受け渡しは値渡しと参照渡しの2種類がある。 また、その違いが明確な言語として、C# や C++ が挙げられる。

値渡し
値をコピーして渡す。２つの変数は値が同じだけで別物。
参照渡し
参照値を渡す。２つの変数が連動する。一心同体の状態。

「a = 10」⇒「b = a」の例


整数が値型でかつ値渡しをする場合
a = 10
b = a


整数が参照型でかつ値渡しをする場合
a = 10
b = a


整数が値型でかつ参照渡しをする場合
a = 10
b = a


整数が参照型でかつ参照渡しをする場合
a = 10

Pythonの場合

Python の型は全てが参照型であり、値の渡し方は全てが値渡しである。

メモリのどこに値が格納されているかを返すid関数を用いて、Pythonでのメモリ管理を調べることができる。
※変数が格納されているアドレスではなく、値が格納されているアドレスを返すことに注意。 
			
num1 = 10
num2 = num1

print(id(num1))   # 実行環境によって異なる
print(id(num2))   # 実行環境によって異なる

print(id(num1) == id(num2))   # True
num2 = 20
print(id(num1) == id(num2))   # False
